## Critical Gaps Requiring Action

### 1. INT-01 — Query/RAG flow broken (HIGH)

`dashboard/src/lib/api.ts` `api.query.ask()` POSTs to `/api/query/ask` (returns JSON) but reads the response body as an SSE stream. No SSE `data:` lines exist in a JSON response, so `QueryView.svelte` always displays an empty assistant message.

**Fix:** In `api.ts`, change the ask URL from `/api/query/ask` → `/api/query/ask/stream`.

### 2. INT-02 — Event search shape mismatch (HIGH)

`EventsView.svelte` calls `res.results.map(r => r.event)` but `/api/events/search` returns `{events:[...], total:N}`. This causes a `TypeError` crash on the search results path.

**Fix:** Update backend `/api/events/search` response to `{results:[{event, score}]}` or update dashboard to read `res.events`.

### 3. P11-T02 — Caddy digest unpinned (DEFERRED)

`docker-compose.yml` uses `caddy:2.9-alpine` without a sha256 digest. Deferred when Docker Desktop was unavailable.

**Fix (single human action):**
```
docker pull caddy:2.9-alpine
docker inspect caddy:2.9-alpine --format '{{index .RepoDigests 0}}'
# Update docker-compose.yml line to: image: caddy:2.9-alpine@sha256:<digest>
```

### 4. INT-04 — SettingsView not routed (MEDIUM)

`SettingsView.svelte` (operator management, key rotation) exists but is not imported or routed in `App.svelte`. Operators can only be managed via raw API.

**Fix:** Add `'settings'` to nav items and route in `App.svelte`.

### 5. P22 — Human UI verification needed (MEDIUM)

Phase 22 passed automated checks (10/10) but flagged 3 items requiring live frontend:
- Advisory banner has no dismiss button (live DOM check)
- Settings > System tab loads model-status card on activation
- Confidence badge colour thresholds render correctly
