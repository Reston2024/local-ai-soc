<task type="auto">
  <name>Task 2: Write scripts/seed_siem_data.py</name>
  <files>scripts/seed_siem_data.py</files>
  <action>
    Write a standalone async script. Implement the following:

    ```python
    #!/usr/bin/env python
    """
    Seed the DuckDB/Chroma stores with up to MAX_ROWS rows from the
    darkknight25/Advanced_SIEM_Dataset Hugging Face dataset.

    Usage:
        uv run python scripts/seed_siem_data.py            # ingest
        uv run python scripts/seed_siem_data.py --dry-run  # validate without writing
        uv run python scripts/seed_siem_data.py --limit 100

    This script is idempotent: rows with duplicate event_ids are silently skipped
    (DuckDB PRIMARY KEY constraint).
    """
    ```

    **Implementation requirements:**

    1. **Argument parsing** — use argparse with `--dry-run` (bool flag) and `--limit` (int, default 500).

    2. **HF dataset loading** — use `datasets.load_dataset()` with:
       - `path="darkknight25/Advanced_SIEM_Dataset"`
       - `split="train"` (the dataset has only a train split)
       - `streaming=True` (avoid downloading full dataset)
       - `trust_remote_code=False` (security: no arbitrary code execution)
       - Take the first `--limit` rows using `itertools.islice(ds, limit)`

    3. **Field normalisation** — map HF columns to NormalizedEvent. The dataset columns vary; write a `_normalise_row(row: dict) -> NormalizedEvent` function:
       - `event_id`: `str(uuid.uuid4())`
       - `timestamp`: parse `row.get("Timestamp") or row.get("timestamp")`, fall back to `datetime.utcnow()`
       - `source_type`: `"hf_siem_seed"`
       - `hostname`: `row.get("Source IP") or row.get("hostname") or row.get("Host") or None`
       - `username`: `row.get("User") or row.get("username") or None`
       - `process_name`: `row.get("Process") or row.get("process") or None`
       - `event_type`: `row.get("Event Type") or row.get("event_type") or "siem_event"`
       - `severity`: normalise `row.get("Severity") or "low"` to lowercase; map "informational"/"info" → "low"; anything not in {"low","medium","high","critical"} → "low"
       - `raw_event`: `json.dumps(row, default=str)`
       - `ingested_at`: `datetime.utcnow()`

    4. **Ingestion** — in `--dry-run` mode: print the first 3 normalised events and total count, then exit 0 without writing to any store. In normal mode:
       - Initialise DuckDBStore, ChromaStore from `settings.DATA_DIR`
       - Start DuckDB write worker (`asyncio.ensure_future(store.start_write_worker())`)
       - Initialise schemas
       - Call IngestionLoader.ingest_events(batch) in batches of 50
       - Print a summary: "Seeded N events from HF SIEM dataset"
       - Properly shut down stores on exit (cancel write worker, close connections)

    5. **Error handling** — if the HF dataset is unavailable (network error, dataset moved), print a clear error and exit 1. Never crash silently.

    6. **Entry point** — `if __name__ == "__main__": asyncio.run(main())`

    The script must work from the project root: `uv run python scripts/seed_siem_data.py --dry-run`
