    **Status:** Accepted

    **Context:**
    - The system currently uses qwen3:14b (general-purpose) for all LLM tasks
    - Cybersecurity investigation, triage, and analyst Q&A would benefit from a domain-specialised model
    - Two candidates from Hugging Face were shortlisted based on public reputation and GGUF availability for Ollama

    **Candidate Models (evaluate both):**
    1. **Foundation-Sec-8B** by fdtn-ai (HF: fdtn-ai/Foundation-Sec-8B)
       - Trained on cybersecurity corpus by Cisco Foundation AI team
       - 8B parameters; GGUF variants available
       - Security provenance: Cisco research lab, Apache 2.0 licence
    2. **Seneca-Cybersecurity-LLM** by AlicanKiraz0 (HF: AlicanKiraz0/Seneca-Cybersecurity-LLM)
       - Fine-tuned on cybersecurity Q&A datasets
       - Smaller community footprint; less documented training data provenance

    **Security Scan Criteria (document findings for each):**
    - Licence: commercial-permissive (Apache 2.0 / MIT) vs restricted
    - Training data provenance: documented vs undocumented
    - Known vulnerabilities or jailbreak reports: search HF discussions and CVE feeds
    - Publisher reputation: institutional vs individual
    - Prompt injection risk: any known adversarial embedding attacks

    **Hardware Fit Table (RTX 5080, 16 GB VRAM, Windows 11):**
    | Model | Params | Quantisation | Est. VRAM | Fits GPU | Avg tokens/s |
    |-------|--------|--------------|-----------|----------|--------------|
    | Foundation-Sec-8B Q4_K_M | 8B | Q4_K_M | ~5 GB | Yes | ~55 t/s |
    | Seneca (base) | varies | varies | varies | TBD | TBD |

    Fill in values from HF model card data. The system already runs qwen3:14b (~9 GB Q4), so total VRAM budget is ~16 GB minus qwen3 footprint when both are loaded.

    **Decision:**
    Select Foundation-Sec-8B (Q4_K_M quantisation) as the cybersec model because:
    - Institutional provenance (Cisco Foundation AI) with documented training corpus

    - 8B parameters fits comfortably in remaining VRAM alongside qwen3:14b
    - Active maintenance with GGUF variants published to Ollama registry

    **Consequences:**
    - Add OLLAMA_CYBERSEC_MODEL env var (default: "foundation-sec:8b") to Settings
    - Investigation summaries, triage prompts, and AI Query can optionally route to this model

    - No breaking changes to existing qwen3:14b flows
