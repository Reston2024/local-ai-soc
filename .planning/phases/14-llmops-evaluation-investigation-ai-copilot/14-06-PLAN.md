# Phase 14 Plan 06 — Gap Closure: Eval Harness Dry-Run DuckDB Lock

## Problem

`uv run python scripts/eval_models.py --dry-run --limit 5` throws:

```
duckdb.duckdb.IOException: IO Error: File is already open in <process>
```

**Root cause:** `scripts/eval_models.py` line 274 calls `DuckDBStore(settings.DATA_DIR)`
unconditionally before checking the `--dry-run` flag. The `DuckDBStore` constructor opens
an exclusive write connection to `data/events.duckdb`. When the backend is running, it already
holds that lock. The dry-run guard at line 151 only skips the Ollama HTTP call — it does not
prevent DuckDB initialization.

Note: Even `duckdb.connect(read_only=True)` will fail while the backend holds the write lock.
The correct fix for dry-run mode is to bypass DuckDB entirely and use synthetic placeholder rows.

## Fix Strategy

In `--dry-run` mode:
1. Skip `DuckDBStore` initialization entirely.
2. Generate `args.limit` synthetic `Row` tuples (with realistic-looking dummy data) to feed
   into the eval loop, instead of reading from `normalized_events`.
3. The rest of the eval loop (`evaluate_row` returning early at line 151, `write_results_jsonl`,
   and the markdown table print) all work unchanged.

This satisfies the UAT expectation:
- No DuckDB lock conflict ✓
- Prints a 4-row markdown table (2 models × 2 prompt types summary) ✓
- Creates `data/eval_results.jsonl` with `args.limit × 4` placeholder entries ✓

# After: args = parser.parse_args()

output_path = Path(_PROJECT_ROOT) / "data" / "eval_results.jsonl"
output_path.parent.mkdir(parents=True, exist_ok=True)

if args.dry_run:
    # Bypass DuckDB entirely — generate synthetic placeholder rows
    DRY_RUN_ROW = (
        "dry-run-id",       # event_id
        "process_create",   # event_type
        "WORKSTATION-01",   # hostname
        "cmd.exe",          # process_name
        "cmd.exe /c whoami",# command_line
        "high",             # severity

    )
    rows = [DRY_RUN_ROW] * args.limit
else:
    # Initialize DuckDB store (requires exclusive write lock)
    duckdb_store = DuckDBStore(settings.DATA_DIR)
    worker_task = duckdb_store.start_write_worker()
    await duckdb_store.initialise_schema()
    try:
        sql = (
            "SELECT event_id, event_type, hostname, process_name, command_line, "
            "severity, attack_technique "
            "FROM normalized_events LIMIT ?"
        )
        rows = await duckdb_store.fetch_all(sql, [args.limit])
    finally:
        worker_task.cancel()
        try:
            await worker_task
        except asyncio.CancelledError:
            pass
    if not rows:
        print("[!] No rows found in normalized_events. Run seed_siem_data.py first.")
        return
