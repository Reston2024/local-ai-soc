Create backend/services/reranker_client.py.

Add a module-level docstring: "Reranker client. Gracefully degrades to passthrough when RERANKER_URL is empty, RERANKER_ENABLED=False, or the service is unreachable."

Implement:

```python
async def rerank_passages(query: str, passages: list[str], top_k: int = 5) -> list[str]:
```

Logic:
- If settings.RERANKER_URL is empty or settings.RERANKER_ENABLED is False:
  - Log at DEBUG level "reranker passthrough mode active"
  - Return passages[:top_k] (no HTTP call)
- Otherwise: POST to f"{settings.RERANKER_URL}/rerank" with payload:
  {"query": query, "passages": passages, "top_k": top_k}
  - Use httpx.AsyncClient as async context manager with timeout:
    httpx.Timeout(connect=2.0, read=8.0, write=4.0, pool=2.0)
  - Parse response: extract ranked[].passage in order, return as list[str]
- On ANY exception (TimeoutException, ConnectError, HTTPStatusError, Exception):
  - Log at WARNING level with the error message
  - Return passages[:top_k] (graceful degradation — never raises)

Use httpx.AsyncClient as a fresh context manager per call (matches the SpiderFoot/TheHive client pattern). Import settings from backend.core.config and get_logger from backend.core.logging.

Log at DEBUG when passthrough; WARNING when a live call fails.

async def query_with_rerank(
    self,
    collection_name: str,
    query_text: str,
    query_embeddings: list[list[float]],
    n_results: int = 20,
    where: dict | None = None,
    top_k: int | None = None,

Implementation:
1. Call `results = await self.query_async(collection_name, query_embeddings, n_results=n_results, where=where)`
2. Extract documents: `documents = results["documents"][0]` (first query's results)
3. Import rerank_passages lazily inside the method to avoid circular imports:
   `from backend.services.reranker_client import rerank_passages`
4. Import settings lazily: `from backend.core.config import settings`
5. Call: `reranked = await rerank_passages(query_text, documents, top_k=top_k or settings.RERANKER_TOP_K)`
6. Reorder the result dict to match the reranked passage order (align ids, distances, metadatas to match document order)
7. Return dict with same shape as Chroma result but truncated/reordered to match reranked order

The existing query() and query_async() callers throughout the codebase are unaffected.
