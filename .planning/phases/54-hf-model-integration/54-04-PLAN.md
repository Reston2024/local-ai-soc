<task type="auto">
  <name>Task 3: Write scripts/rebuild_chroma.py</name>
  <files>scripts/rebuild_chroma.py</files>
  <action>
Create scripts/rebuild_chroma.py as a standalone Python script (NOT a FastAPI endpoint — imports stores directly without using the app factory).

The script structure:
1. Load settings: `from backend.core.config import settings`
2. Import `from backend.stores.chroma_store import ChromaStore`
3. Import `from backend.stores.duckdb_store import DuckDBStore` — DuckDBStore owns fetch_all(), not ChromaStore
4. Import `from backend.services.ollama_client import OllamaClient`
5. Parse `--dry-run` CLI flag via argparse

Main logic (wrapped in `asyncio.run(main())`):
- For each collection in `["soc_evidence", "feedback_verdicts"]`:
  - In dry-run: log current count via `chroma_store.count(name)` and skip deletion
  - In full run: call `chroma_store.delete_collection(name)` if it exists; catch NotFoundError and log "collection {name} did not exist — creating fresh"
- Re-embed soc_evidence:
  - `rows = await asyncio.to_thread(duckdb_store.fetch_all, "SELECT event_id, raw FROM events LIMIT 10000")`
  - Batch embed using `await ollama_client.embed_batch([r["raw"] for r in rows])` in batches of 100
  - Call `chroma_store.add_documents(...)` for each batch
- Re-embed feedback_verdicts:
  - Query SQLite for feedback rows (use sqlite3 directly with settings.SQLITE_PATH)
  - Embed content fields, upsert to "feedback_verdicts" collection
- Log final counts: `chroma_store.count("soc_evidence")` and `chroma_store.count("feedback_verdicts")`
- Print completion summary: "Rebuild complete: soc_evidence={N} records, feedback_verdicts={M} records"

Use asyncio.run() as the top-level entry point. DuckDB reads use `await asyncio.to_thread(store.fetch_all, sql)` pattern. Add structured logging via `backend.core.logging.get_logger(__name__)`.
  </action>
