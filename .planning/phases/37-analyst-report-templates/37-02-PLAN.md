PIR data sources (from RESEARCH.md):
- investigation_cases: get_investigation_case(case_id) — must have case_status='closed' (blank if not closed or not found)
- detections: sqlite._conn SELECT * FROM detections WHERE investigation_id = ? LIMIT 50
- detection_techniques JOIN attack_techniques: SELECT dt.detection_id, at.technique_id, at.name FROM detection_techniques dt JOIN attack_techniques at ON dt.technique_id = at.technique_id WHERE dt.detection_id IN (subquery)
- playbook_runs: sqlite._conn SELECT * FROM playbook_runs WHERE investigation_id = ? ORDER BY started_at DESC
- closed case selector built from list_investigation_cases called with status filter

TI Bulletin data sources (from RESEARCH.md):
- attack_groups: SELECT name, group_id, aliases FROM attack_groups WHERE name = ? (exact) ELSE LOWER(name) LIKE LOWER(?)||'%'
- attack_group_techniques JOIN attack_techniques: SELECT at.technique_id, at.name, at.tactic FROM attack_group_techniques agt JOIN attack_techniques at ON agt.technique_id = at.technique_id WHERE agt.stix_group_id = (SELECT stix_id FROM attack_groups WHERE name = ?)
- ioc_store: SELECT ioc_value, ioc_type, confidence, feed_source, malware_family FROM ioc_store WHERE LOWER(actor_tag) LIKE LOWER(?)||'%' AND ioc_status = 'active' ORDER BY confidence DESC LIMIT 100
  (Pitfall 3: actor_tag fuzzy match — use LIKE with '%' suffix for name matching)
- asset_store: SELECT ip, hostname, tag, risk_score FROM asset_store ORDER BY risk_score DESC LIMIT 20

Severity Reference data sources: NONE — pure static HTML.

Known severity levels to hardcode:
  Critical: Active exploitation underway, confirmed compromise, data exfiltration
  High: Strong indicator of compromise, lateral movement, privilege escalation
  Medium: Suspicious activity requiring investigation, policy violation
  Low: Informational anomaly, benign likely, review recommended
  Informational: Telemetry/audit log entry, no action required

Known open gaps (4 hardcoded from docx template):
  1. No email gateway integration — phishing delivery blind spot
  2. No EDR agent on legacy Windows XP/2003 assets — endpoint telemetry gap
  3. Cloud workload visibility limited to firewall north-south flows only
  4. Zeek SSH key fingerprinting not yet correlated to known threat-actor infrastructure

Confidence scoring rubric (ioc_store scale 0-100):
  90-100: High-fidelity feed (Feodo, CISA KEV), confirmed malicious
  60-89: Moderate confidence (ThreatFox crowd-sourced)
  30-59: Low confidence, enrichment pending
  0-29: Indicative only, treat as noise unless corroborated

    1. Fetch actor info + techniques + IOCs in a single asyncio.to_thread(_fetch_ti_data, conn, actor_name):
       ```python
       def _fetch_ti_data(conn, actor_name: str) -> dict:
           # Actor lookup with fallback fuzzy match
           actor_row = conn.execute("SELECT * FROM attack_groups WHERE name = ?", [actor_name]).fetchone()
           if not actor_row:
               actor_row = conn.execute("SELECT * FROM attack_groups WHERE LOWER(name) LIKE LOWER(?)||'%'", [actor_name]).fetchone()
           # Techniques
           if actor_row:
               stix_id = actor_row[0]  # stix_id column
               techniques = conn.execute("SELECT at.technique_id, at.name, at.tactic FROM attack_group_techniques agt JOIN attack_techniques at ON agt.technique_id = at.technique_id WHERE agt.stix_group_id = ?", [stix_id]).fetchall()
           # IOCs with fuzzy actor_tag match
           ioc_rows = conn.execute("SELECT ioc_value, ioc_type, confidence, feed_source, malware_family FROM ioc_store WHERE LOWER(actor_tag) LIKE LOWER(?)||'%' AND ioc_status = 'active' ORDER BY confidence DESC LIMIT 100", [actor_name]).fetchall()
           # Assets
           asset_rows = conn.execute("SELECT ip, hostname, tag, risk_score FROM asset_store ORDER BY risk_score DESC LIMIT 20").fetchall()
           return {actor_row, techniques, ioc_rows, asset_rows}
       ```
    2. Triage: asyncio.to_thread(stores.sqlite.get_latest_triage)
