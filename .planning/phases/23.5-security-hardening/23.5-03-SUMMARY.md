## What Was Built

### Task 1: _normalize_for_scrub helper + hardened _scrub_injection

Added `_normalize_for_scrub(text)` to `ingestion/normalizer.py` which:
1. Applies `unicodedata.normalize("NFC", text)` to collapse Unicode homoglyphs
2. Attempts base64 decoding with a discriminating heuristic (input must be >=16 chars, all base64 alphabet characters, decoded output must have >70% text characters) to avoid false positives on normal text
3. Appends decoded text to the normalized string so the injection regex matches against both representations

Updated `_scrub_injection` to call `_normalize_for_scrub` first, then apply `_INJECTION_PATTERNS.sub`.

Activated `test_base64_bypass_scrubbed` in `tests/security/test_injection_hardening.py` (removed `@pytest.mark.skip` and `assert False` stub).

### Task 2: EVIDENCE to system turn + chat scrubbing

Updated `analyst_qa.build_prompt` to return `tuple[str, str]` instead of `str`:
- `system_addition`: evidence blocks wrapped in `--- BEGIN/END EVIDENCE CONTEXT ---` for the system prompt
- `user_turn`: only `"Question: {question}"` with no evidence in the user turn

This separates the LLM trust domains — attacker-controlled data in evidence fields cannot influence the user-turn message flow.

Updated `tests/eval/test_analyst_qa_eval.py` to unpack the tuple and pass `analyst_qa.SYSTEM + system_addition` as the system argument.

Updated `backend/api/chat.py`:
- Added `from ingestion.normalizer import _scrub_injection`
- Added `safe_question = _scrub_injection(body.question)` before prompt construction
- `prompt` uses `safe_question`; `insert_chat_message` stores original `body.question` for audit trail

Activated `test_chat_question_scrubbed` in `tests/security/test_injection_hardening.py`.
