## Decisions Made

- `dev-only-bypass` is the explicit allowlisted weak token for local dev without .env — all other tokens <32 chars rejected at startup
- `LEGACY_TOTP_SECRET=""` default makes legacy path opt-in — deployment requires explicit `.env` configuration to use the legacy path at all
- `isinstance(x, str)` guard before `.strip()` prevents MagicMock from being treated as a valid non-empty secret in tests that use `patch("backend.core.auth.settings")` with MagicMock
- Tests that were relying on the legacy token fallback for auth bypass were updated to use `dependency_overrides[verify_token]` — this is the correct pattern for tests that test non-auth functionality

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed MagicMock bypass in LEGACY_TOTP_SECRET check**
- **Found during:** Task 2 (test run after auth.py modification)
- **Issue:** `getattr(settings, "LEGACY_TOTP_SECRET", "").strip()` returns a truthy MagicMock when settings is a MagicMock (from tests using `patch("backend.core.auth.settings")`). This caused the legacy path to proceed to TOTP verification with a MagicMock secret, crashing pyotp.
- **Fix:** Changed to `_raw_secret = getattr(settings, "LEGACY_TOTP_SECRET", ""); legacy_secret = _raw_secret.strip() if isinstance(_raw_secret, str) else ""`
- **Files modified:** `backend/core/auth.py`
- **Committed in:** 2c3efc9 (Task 2 commit)

**2. [Rule 1 - Bug] Updated test_compliance_export.py and test_grounding.py to use dependency_overrides**
- **Found during:** Task 2 (full suite run revealed 9 failures in compliance and grounding tests)
- **Issue:** These tests patched `backend.core.auth.settings` with a MagicMock and sent a short `AUTH_TOKEN` via Bearer header. The legacy path is now intentionally disabled (requires LEGACY_TOTP_SECRET), so all requests using the legacy token fallback returned 401.
- **Fix:** Updated fixtures in both test files to use `app.dependency_overrides[verify_token] = lambda: _ctx` — the correct pattern for tests that test non-auth functionality.
- **Files modified:** `tests/unit/test_compliance_export.py`, `tests/eval/test_grounding.py`
- **Committed in:** 2c3efc9 (Task 2 commit)

**3. [Rule 1 - Bug] Updated test_auth.py legacy path tests to reflect new behavior**
- **Found during:** Task 2 (test_auth.py showed 3 failures after auth.py change)
- **Issue:** `test_valid_token_passes`, `test_legacy_token_fallback`, `test_state_injection` relied on legacy path working with just AUTH_TOKEN match. Now gated behind TOTP.
- **Fix:** test_valid_token_passes now tests via operator table path; test_legacy_token_fallback asserts 401 with "legacy" in detail; test_state_injection uses operator table path.
- **Files modified:** `tests/unit/test_auth.py`
- **Committed in:** 2c3efc9 (Task 2 commit)
