Threat scoring model (LOCKED — additive, 0-100):
```
base_score = 0
+ suricata_severity_points: critical=40, high=30, medium=20, low=10, info/none=0
+ sigma_hit: +20 if alert.rule matches UUID regex (sigma-sourced)
+ recurrence: +10 if same host/IP (alert.event_id used to look up event) seen >= 3 times in events list
+ graph_connectivity: +10 if graph_data is not None and has >= 3 edges for the host/IP
score = min(score, 100)
```

UUID regex pattern: `^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$`

IMPORTANT — graph_connectivity performance trap:
- Do NOT call build_graph() inside score_alert — this is O(n²) for batch ingestion
- Accept graph_data: dict | None = None
- When graph_data is None (default), skip the +10 entirely (score 0 for that component)
- This avoids quadratic ingest cost while keeping the interface correct
