**docs/decision-log.md** â€” append a Phase 5 section with these decisions:

1. **Suricata EVE `dest_ip` â†’ `dst_ip` mapping** â€” EVE uses `dest_ip`/`dest_port` (Snort convention), not `dst_ip`/`dst_port`. Normalized schema uses `dst_ip`. Parser explicitly maps `data.get("dest_ip")` â†’ `dst_ip`. Failing to do this leaves dst_ip as None for all Suricata events and suppresses graph IP nodes.

2. **Suricata severity is inverted** â€” EVE severity 1 = critical (highest priority), 4 = low (lowest). This is the Snort convention, opposite of common intuition. Mapping: `{1: "critical", 2: "high", 3: "medium", 4: "low"}`. Tests assert severity=1 produces "critical".

3. **Threat scoring is additive integer 0â€“100** â€” Simple deterministic formula chosen over ML model (deferred). Components: suricata severity (40/30/20/10) + sigma hit (+20) + recurrence (+10) + graph connectivity (+10). Capped at 100. No floating-point, no weights to tune.

4. **score_alert graph_connectivity defaults to None/skip** â€” Calling `build_graph()` inside `_store_event()` for every event would be O(nÂ˛) for batch ingestion. Accept `graph_data: dict | None = None`; when None, skip the +10 graph component. Correct tradeoff: connectivity scoring available for on-demand calls, skipped for batch ingest.

5. **Deferred import for threat_scorer + attack_mapper in routes.py** â€” Module-level imports fail at backend startup if modules are absent (during incremental development). Deferred import with try/except ImportError inside `_store_event()` follows the sigma_loader.py pattern â€” graceful degradation.

6. **ATT&CK tagging is a static 5-entry lookup** â€” Full ATT&CK coverage (700+ techniques) deferred. Static mapping covers the five most actionable categories. Returns `[]` for unmapped events â€” no guessing. Documented clearly in `attack_mapper.py` module docstring.

7. **Suricata Docker service is scaffolded with Windows blocker** â€” `jasonish/suricata` requires `--net=host` and Linux kernel capabilities (`net_admin`, `net_raw`, `sys_nice`) that Docker Desktop for Windows (WSL2 backend) does not provide. NFQUEUE kernel module unavailable. Scaffold added to docker-compose.yml with documented blocker; validation uses `fixtures/suricata_eve_sample.ndjson` only.

8. **alert event_type = alert.signature, not "alert"** â€” For Suricata alert events, `event_type` is set to the rule signature string (e.g., "ET MALWARE CobaltStrike Beacon"), not the generic "alert". This allows Sigma rules and enrichment to match on the specific signature text.
