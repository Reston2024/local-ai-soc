<objective>
Add Caddy request_body size limits to guard against oversized evidence uploads exhausting FastAPI memory.

Purpose: Block accidental multi-GB uploads at the proxy layer — before Python allocates memory for the body. Caddy returns 413 for oversized requests without involving FastAPI at all.

Output: Caddyfile updated with two-tier limits (100MB for file uploads, 10MB for all other API requests). The SSE streaming handler for /api/query/* is unaffected.
</objective>

Required updated order:

3. handle /api/*               — 10MB body limit (general fallback)
4. handle /health
5. handle /app/*
6. handle /

Caddy request_body syntax (v2.10+ confirmed installed as v2.11.1):
```
handle /api/ingest/file {
    request_body {
        max_size 100MB
    }
    reverse_proxy host.docker.internal:8000 {
        header_up Host {host}
        header_up X-Real-IP {remote_host}
        header_up X-Forwarded-For {remote_host}
        header_up X-Forwarded-Proto {scheme}
    }
}
```

PITFALL: request_body must be ALONGSIDE reverse_proxy, not nested INSIDE it.
PITFALL: The specific /api/ingest/file handler must come BEFORE the generic /api/* handler.
