  <behavior>
    - POST /api/playbooks/{playbook_id}/run/{investigation_id}: creates run with status="running", steps_completed=[], returns 201 with run dict; returns 404 if playbook not found
    - PATCH /api/playbook-runs/{run_id}/step/{step_n}: body is PlaybookRunAdvance; validates step_n is within playbook steps range; appends step result to steps_completed as {"step_number": step_n, "outcome": outcome, "analyst_note": analyst_note, "completed_at": ISO8601}; if step_n equals total steps count, sets status="completed" and completed_at; returns updated run; returns 404 if run not found; returns 409 if run already completed/cancelled
    - PATCH /api/playbook-runs/{run_id}/cancel: sets status="cancelled", completed_at=now; returns updated run; 404 if not found; 409 if already completed
    - GET /api/playbook-runs/{run_id}: returns single run or 404
    - GET /api/playbook-runs/{run_id}/stream: SSE endpoint — fetches current run state, emits one "run_state" event with full run dict, then emits "done" event; this is a snapshot stream (plan 03 frontend polls for updates and this endpoint delivers current state snapshot for initial load)
    - Unit tests cover: start run 404 on missing playbook, advance step appends to steps_completed, advance last step sets completed, cancel sets cancelled, double-complete returns 409
  </behavior>
  <action>
    Add the following endpoints to backend/api/playbooks.py (extend the existing router — do NOT create a new file):

    1. POST /api/playbooks/{playbook_id}/run/{investigation_id}
       - Fetch playbook via asyncio.to_thread(stores.sqlite.get_playbook, playbook_id)
       - Return 404 JSONResponse if None
       - Build run dict: run_id=str(uuid4()), playbook_id, investigation_id, status="running", started_at=utcnow_iso(), completed_at=None, steps_completed=[], analyst_notes=""
       - asyncio.to_thread(stores.sqlite.create_playbook_run, run_dict)
       - Return JSONResponse(run_dict, status_code=201)

    2. PATCH /api/playbook-runs/{run_id}/step/{step_n} (body: PlaybookRunAdvance)
       - Fetch run; 404 if None
       - 409 if run["status"] in ("completed", "cancelled")
       - Fetch parent playbook to get total step count; steps are 1-indexed
       - Append to steps_completed: {"step_number": step_n, "outcome": body.outcome, "analyst_note": body.analyst_note, "completed_at": utcnow_iso()}
       - If step_n >= len(playbook["steps"]) → set status="completed", completed_at=now
       - asyncio.to_thread(stores.sqlite.update_playbook_run, run_id, {"steps_completed": updated_list, "status": new_status, "completed_at": completed_at})
       - Return updated run as JSONResponse

    3. PATCH /api/playbook-runs/{run_id}/cancel
       - Fetch run; 404 if None; 409 if already completed/cancelled
       - asyncio.to_thread(stores.sqlite.update_playbook_run, run_id, {"status": "cancelled", "completed_at": utcnow_iso()})
       - Return updated run

    4. GET /api/playbook-runs/{run_id}
       - Fetch run; 404 if None; return JSONResponse(run)

    5. GET /api/playbook-runs/{run_id}/stream (SSE snapshot)
       - Fetch run; 404 if None
       - async def event_stream(): yield run_state event, yield done event
       - Return StreamingResponse(event_stream(), media_type="text/event-stream", headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})

    SSE event shape: {"event": "run_state", "run": run_dict} then {"done": True}

    Note: The /api/playbook-runs/* prefix is different from /api/playbooks/* — use a second APIRouter(prefix="/api/playbook-runs") in the same file, or add these routes with explicit paths. Register both routers in main.py.

    Helper: def utcnow_iso() -> str: return datetime.now(timezone.utc).isoformat()
