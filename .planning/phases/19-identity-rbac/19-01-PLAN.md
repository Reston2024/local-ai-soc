Replace backend/core/auth.py with the multi-operator implementation. The new
signature MUST accept `request: Request` as first parameter (FastAPI injects it
automatically; existing route Depends(verify_token) calls are unaffected).

Key implementation rules:
- Import OperatorContext from backend.core.rbac
- Import verify_api_key, _dummy_hash, key_prefix from backend.core.operator_utils
- Lookup order:
  1. prefix = key_prefix(raw)
  2. row = stores.sqlite.get_operator_by_prefix(prefix)  # synchronous, wrap in asyncio.to_thread
  3. If row found AND verify_api_key(raw, row["hashed_key"]) → build OperatorContext, call update_last_seen
  4. If row not found → run verify_api_key(raw, _dummy_hash) (discard result — constant-time dummy)
  5. If no operator found → check hmac.compare_digest(raw, settings.AUTH_TOKEN.strip()) for legacy path
  6. If legacy match → OperatorContext(operator_id="legacy-admin", username="admin", role="admin", totp_verified=True)
  7. If nothing → raise HTTPException(401)
- Set request.state.operator = ctx before returning ctx
- Return type annotation: -> OperatorContext (not None)
- Keep existing ?token= query param fallback for binary downloads
- Keep AUTH_TOKEN empty-string misconfiguration guard (raise 401 before any lookup)

TOTP enforcement: skip for Phase 19-01; the totp_verified field defaults True.
Phase 19-03 will add the TOTP verification step.
