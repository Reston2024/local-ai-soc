## Goal

Thread `audit_id` and `grounding_event_ids` back through `generate()` / `stream_generate()` to the `/ask` and `/ask/stream` API responses, and add `is_grounded` flag, without breaking any existing callers.

## Context

**Current state of `generate()` (ollama_client.py ~line 302):**
- Signature: `generate(self, prompt, system, temperature, model, use_cybersec_model, operator_id, prompt_template_name, prompt_template_sha256, grounding_event_ids) -> str`
- `audit_id = str(uuid4())` is created internally on line ~331 but never returned
- After provenance write, returns `response_text` (str only)
- All callers in `query.py`, `chat.py`, `explain.py` expect a plain `str` return

**Current state of `ask()` (query.py ~line 170):**
- Calls `await ollama.generate(prompt, system=system)` — no grounding_event_ids, no out_context
- Returns `{question, answer, context_event_ids, context_count, citation_verified}`
- Missing: `audit_id`, `is_grounded`

**Current state of `ask_stream()` (query.py ~line 251):**
- Uses `ollama.stream_generate_iter(prompt, system=system)` which has NO provenance write
- Must migrate to `stream_generate()` + `on_token` callback pattern
- Done-event currently: `{"done": True, "context_event_ids": ids}` — add `audit_id`, `is_grounded`

**Open question resolution (from research):** Resolve `operator_id` from `request.state` if available (attribute `operator_id`), otherwise fall back to `"system"`. Do not refactor auth — one-liner fallback.

**Pitfall to avoid:** Do NOT change `generate()` return type from `str`. Use `out_context: dict | None = None` parameter — write `{"audit_id": ..., "grounding_event_ids": [...]}` into it after provenance write. Callers that don't pass `out_context` are unaffected.

In `ask_stream()`:
1. Replace `ollama.stream_generate_iter(prompt, system=system)` usage inside `event_stream()` with `stream_generate()` using an `on_token` callback:
   ```python
   tokens: list[str] = []
   out_ctx: dict = {}

   def on_token(t: str) -> None:
       tokens.append(t)

   await ollama.stream_generate(
       prompt,
       system=system,
       on_token=on_token,
       grounding_event_ids=ids,
       out_context=out_ctx,
   )
   ```
   Note: `stream_generate()` is non-async-generator — it calls `on_token` for each token and returns when complete. The `event_stream()` async generator must yield SSE tokens by collecting them. Look at how `stream_generate()` is defined in ollama_client.py and adapt accordingly. If `stream_generate()` is itself async, await it inside a wrapper. The key is: yield `data: {"token": t}\n\n` for each token, then yield the done event with `audit_id` and `is_grounded`.

   Simplest working pattern — wrap stream_generate in event_stream:
   ```python
   async def event_stream() -> AsyncIterator[str]:
       try:
           tokens_yielded: list[str] = []
           out_ctx: dict = {}

           def _on_token(t: str) -> None:
               tokens_yielded.append(t)

           await ollama.stream_generate(
               prompt,
               system=system,
               on_token=_on_token,
               grounding_event_ids=ids,
               out_context=out_ctx,
           )
           for t in tokens_yielded:
               yield f"data: {json.dumps({'token': t})}\n\n"

           audit_id = out_ctx.get("audit_id")
           is_grounded = len(ids) > 0
           yield f"data: {json.dumps({'done': True, 'context_event_ids': ids, 'audit_id': audit_id, 'is_grounded': is_grounded})}\n\n"
       except Exception as exc:
           yield f"data: {json.dumps({'error': str(exc)})}\n\n"
   ```

   Important: If `stream_generate()` takes a streaming approach that blocks until complete before returning, this pattern works correctly. Verify by reading `stream_generate()` signature in `ollama_client.py` before implementing.
