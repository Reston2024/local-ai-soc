Output: Confirmed (or refreshed) sha256 digest in docker-compose.yml.
</objective>

  <what-built>The previous task checks whether the digest is already pinned. This step is only needed if the digest is MISSING or you want to verify it matches the current upstream image.</what-built>
  <how-to-verify>
    **Only perform these steps if Task 1 reported "DIGEST MISSING" or you want to confirm the digest is current:**

    1. Ensure Docker Desktop is running (check the system tray icon)
    2. Open a terminal and run:
       ```
       docker pull caddy:2.9-alpine
       docker inspect caddy:2.9-alpine --format "{{index .RepoDigests 0}}"
       ```
    3. The output will be: `caddy@sha256:<64-char-hex>`
    4. Copy the sha256 value (64 hex characters after `sha256:`)
    5. Open `docker-compose.yml` and update the image line to:
       ```
       image: caddy:2.9-alpine@sha256:<paste-your-digest-here>
       ```
    6. Save the file

    **If Task 1 already showed "DIGEST PRESENT" with the existing digest `b4e3952...`:**
    - The pin is already in place. You can optionally run the docker inspect command above to confirm the digest still matches the upstream tag.
    - If the digests match → type "approved — existing digest confirmed"
    - If Docker Desktop is not running and you cannot verify → type "approved — existing digest accepted" (the digest was recorded at plan time and is immutable by design)
  </how-to-verify>
